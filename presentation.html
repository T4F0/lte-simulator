    Attach Procedure — Phase 1

    Steps 1–5: Discovery & Authentication

    1

    UE Powers On

    Device initializes radio, reads SIM card (USIM), retrieves IMSI and authentication keys.

    2

    Cell Search

    UE scans frequencies, finds the strongest eNodeB signal, synchronizes timing, and reads the System Information Block (SIB).

    UE measures RSRP/RSRQ → selects best cell
    3

    Attach Request

    UE sends NAS Attach Request containing IMSI, UE capabilities, and requested PDN type to the eNodeB.

    UE → eNodeB : RRC Connection + Attach Request
    4

    Forward to MME

    eNodeB wraps the message in S1-AP and sends it to the MME over the S1 interface.

    eNodeB → MME : Initial UE Message (S1-AP)
    5

    HSS Authentication Query

    MME contacts the HSS via Diameter protocol to fetch authentication vectors for the subscriber's IMSI.

    MME → HSS : Authentication Info Request (Diameter)
    06 / 10

    Attach Procedure — Phase 2

    Steps 6–10: Security & Session

    6

    EPS-AKA Authentication

    MME sends a challenge (RAND, AUTN) to the UE. UE verifies network authenticity and responds with RES. MME compares RES with XRES.

    MME → UE : Auth Request | UE → MME : Auth Response
    7

    NAS Security Mode

    MME and UE negotiate encryption (EEA) and integrity (EIA) algorithms. All further NAS messages are protected.

    Security Mode Command → Security Mode Complete
    8

    Create Session (GTP)

    MME sends Create Session Request to SGW, which forwards to PGW. This establishes the data path (bearer).

    MME → SGW → PGW : Create Session Request (GTP-C)
    9

    IP Address Assigned

    PGW allocates an IP address from its pool and returns it via SGW back to the MME. The default bearer is now established.

    PGW → SGW → MME : IP = 10.20.x.x, QCI = 9
    10

    Attach Complete ✓

    MME sends Attach Accept to UE. UE responds with Attach Complete. Device is now registered and has internet access!

    ✓ Status: ATTACHED | Internet Access: ACTIVE
    07 / 10

    Security

    EPS-AKA Authentication

    Evolved Packet System — Authentication and Key Agreement. The mutual authentication protocol that protects LTE networks.

    🔑

    Shared Secret Key (K)

    Pre-shared between SIM card and HSS. Never transmitted over the air.

    🎲

    RAND (Random Challenge)

    HSS generates random number. UE must produce the correct response using key K.

    🔐

    Mutual Authentication

    UE also verifies the network using AUTN — prevents fake base station attacks.

    🛡️

    KASME (Session Key)

    Derived key used for all further encryption. Changes every session for forward secrecy.

    Authentication Flow:
    HSS generates: RAND, XRES, AUTN, KASME
    → sends Auth Vector to MME
    MME → UE : RAND + AUTN
    UE verifies AUTN using key K
    ✓ Network is genuine
    UE computes RES from RAND + K
    UE → MME : RES
    MME compares RES == XRES
    ✓ Subscriber authenticated

    Mobility

    Cell Handover

    When you move from one tower's coverage to another, the network seamlessly transfers your connection — without dropping the call or session.

    📡
    Tower A Signal weakening
    ⟹ X2 Handover
    📡
    Tower B Stronger signal
    📊

    1. Measurement Report

    UE continuously measures signal strength of neighboring cells and reports to the serving eNodeB.

    📋

    2. Handover Decision

    Source eNodeB decides to handover based on A3 event (neighbor stronger than serving by offset).

    🔄

    3. X2 Handover

    Source eNodeB sends HO Request to target, context is transferred, UE switches. SGW path is updated via MME.

    Error Handling

    Failure Scenarios

    Not every attach attempt succeeds. The LTE network handles errors gracefully with specific rejection causes.

    🚫

    Invalid SIM

    The IMSI on the SIM card is not found in the HSS database. The subscriber is unknown to the network.

    HSS → MME
    IMSI not found
    Cause: #2 IMSI Unknown
    EMM Cause #2
    ⚠️

    Authentication Failure

    The UE failed to produce the correct RES response. Could mean tampered SIM or key mismatch between SIM and HSS.

    UE → MME
    MAC mismatch
    Cause: RES ≠ XRES
    EMM Cause #20
    📡

    Tower Unreachable

    No eNodeB signal detected. Could be caused by outage, out-of-coverage area, or radio interference.

    No cell detected
    RRC Connection Failure
    Cause: No Coverage
    RRC Failure
